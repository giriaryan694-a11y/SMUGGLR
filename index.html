Emoji Smuggler — Variation Selector Steganography

Based on Paul Butler's February 2025 research. Each UTF-8 byte of your secret maps to one of 256 invisible Unicode Variation Selectors (U+FE00–FE0F and U+E0100–E01EF) appended directly after a carrier emoji. The result looks like a single normal emoji but carries an arbitrary hidden payload — survives copy-paste across all platforms.

Encoding Method
Paul Butler's method (recommended). Every UTF-8 byte of your secret → one invisible variation selector. Bytes 0–15 → U+FE00..FE0F  ·  Bytes 16–255 → U+E0100..E01EF. One emoji can carry unlimited bytes. Completely undetectable visually. Survives copy-paste everywhere.
Appends invisible Unicode Tag Block characters (U+E0001 language tag + one char per byte) after the emoji. The same technique found in AI watermarking systems. Zero visual difference on any renderer.
Encodes each bit as U+200D ZWJ (= 1) or U+200C ZWNJ (= 0) between copies of the carrier emoji. Requires 1 emoji per bit — keep secrets very short (3–5 chars). The output chain looks like a sequence of the same emoji repeated.
